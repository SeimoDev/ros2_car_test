# 项目2四个实验运行文档

适用机器：`nx@10.115.6.72`

工作区：

```bash
~/ros2_car_test
```

使用前先执行：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
```

当前这台车已验证的设备口：

- 电机串口：`/dev/ttyUSB0`
- 激光雷达串口：`/dev/ttyUSB1`

## 实验1：里程计与运动模型

启动：

```bash
ros2 launch robot_bringup experiment1_odom.launch.py
```

另开一个终端控制小车：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
ros2 run jetson_car_demo teleop_keyboard --ros-args -p output_topic:=/cmd_vel
```

常用查看命令：

```bash
ros2 topic list
ros2 topic echo /odom
ros2 topic echo /tf
```

说明：

- 本实验会启动电机节点和状态发布节点
- 状态发布节点会发布 `/odom` 和 `/tf`
- 可配合 `rviz2` 观察轨迹

## 实验2：多传感器数据读取

启动：

```bash
ros2 launch robot_bringup experiment2_sensors.launch.py
```

查看传感器数据：

```bash
ros2 topic echo /odom
ros2 topic echo /imu/data_raw
ros2 topic echo /battery_voltage
```

查看话题信息：

```bash
ros2 topic info /odom
ros2 topic info /imu/data_raw
ros2 topic info /battery_voltage
```

说明：

- `/odom`：里程计
- `/imu/data_raw`：IMU 模拟状态
- `/battery_voltage`：电池电压模拟状态

## 实验3：速度闭环控制初步

启动主链路：

```bash
ros2 launch robot_bringup project2_speed_control.launch.py
```

另开一个终端发送原始速度：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
ros2 run jetson_car_demo teleop_keyboard --ros-args -p output_topic:=/cmd_vel_raw
```

查看 PID 参数：

```bash
ros2 param get /heading_pid p
ros2 param get /heading_pid i
ros2 param get /heading_pid d
```

动态修改 PID 参数：

```bash
ros2 param set /heading_pid p 1.5
ros2 param set /heading_pid i 0.0
ros2 param set /heading_pid d 0.1
```

说明：

- `teleop_keyboard` 发布 `/cmd_vel_raw`
- `heading_pid` 订阅 `/cmd_vel_raw` 和 `/odom`
- `heading_pid` 输出修正后的 `/cmd_vel`
- 电机节点最终订阅 `/cmd_vel`

## 实验4：综合实践与展示

启动避障实验主链路：

```bash
ros2 launch robot_bringup project2_demo.launch.py
```

另开一个终端发送原始速度：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
ros2 run jetson_car_demo teleop_keyboard --ros-args -p output_topic:=/cmd_vel_raw
```

查看关键话题：

```bash
ros2 topic list
ros2 topic echo /scan
ros2 topic echo /cmd_vel_raw
ros2 topic echo /cmd_vel
```

说明：

- 雷达驱动发布 `/scan`
- `teleop_keyboard` 发布 `/cmd_vel_raw`
- `safety_stop` 订阅 `/scan` 和 `/cmd_vel_raw`
- `safety_stop` 输出安全过滤后的 `/cmd_vel`
- 当前安全距离阈值默认是 `0.5` 米

## 实时图形化查看激光雷达

一键启动雷达和图形化窗口：

```bash
ros2 launch robot_bringup lidar_view.launch.py
```

如果需要手动指定雷达口：

```bash
ros2 launch robot_bringup lidar_view.launch.py serial_port:=/dev/ttyUSB1
```

说明：

- 会同时启动雷达驱动和 `rviz2`
- 默认使用仓库自带的雷达 RViz 配置
- 默认固定坐标系是 `laser`
- 必须在带桌面的本机图形环境运行，纯文本 SSH 终端不能直接弹出窗口

## 键盘控制说明

运行：

```bash
ros2 run jetson_car_demo teleop_keyboard --ros-args -p output_topic:=/cmd_vel_raw
```

按键：

- `i` 前进
- `,` 后退
- `j` 左转
- `l` 右转
- `u o m .` 组合方向
- `q/z` 整体加减速
- `w/x` 线速度加减
- `e/c` 角速度加减
- `k` 停止
- `Ctrl+C` 退出

## 常见问题

### 1. 提示 `Package not found`

先执行：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
```

### 2. 雷达启动超时

当前这台车雷达口应使用：

```bash
/dev/ttyUSB1
```

单独测试：

```bash
ros2 launch robot_bringup lidar.launch.py serial_port:=/dev/ttyUSB1
```

### 2.1 `rviz2` 只有空网格，看不到雷达数据

优先直接使用：

```bash
ros2 launch robot_bringup lidar_view.launch.py serial_port:=/dev/ttyUSB1
```

如果还是只有网格，按下面顺序检查：

1. 确认雷达节点真的在发 `/scan`

```bash
ros2 topic info /scan
```

正常时至少应看到：

- `Type: sensor_msgs/msg/LaserScan`
- `Publisher count: 1`

2. 单独启动雷达，观察终端有没有报错

```bash
ros2 launch robot_bringup lidar.launch.py serial_port:=/dev/ttyUSB1
```

常见错误：

- `SL_RESULT_OPERATION_TIMEOUT`
- 串口打不开
- 串口口号写错

3. 在 RViz 里确认显示配置

- `Fixed Frame` = `laser`
- 已添加 `LaserScan`
- `Topic` = `/scan`

4. 确认当前是在图形桌面环境，不是纯 SSH 文本终端

### 3. 电机不动

检查：

```bash
ros2 launch robot_bringup motor.launch.py
```

确认电机口是：

```bash
/dev/ttyUSB0
```

### 4. 旧节点残留导致冲突

先关闭旧终端，必要时重新开一个 SSH 终端再启动实验。
