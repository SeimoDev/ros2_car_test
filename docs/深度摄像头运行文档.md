# 深度摄像头运行文档

## 设备信息

- 设备型号：Orbbec Astra Pro
- 远端设备：`nx@10.115.6.72`
- ROS 发行版：`foxy`
- 工作目录：`~/ros2_car_test`

## 当前结论

- 深度摄像头已经在官方 Orbbec SDK 层打通。
- 官方样例 `hello_orbbec` 可以稳定识别设备，序列号为 `15122510407`。
- 官方样例 `depth_viewer` 可以启动深度流。
- 官方样例 `QuickStart` 默认同时打开彩色和深度流，在这台设备上会因为彩色流模式协商失败而退出，所以当前不要用它做验证。

## 一键查看深度画面

先在远端终端执行：

```bash
source /opt/ros/foxy/setup.bash
source ~/ros2_car_test/install/setup.bash
ros2 launch robot_bringup depth_camera_view.launch.py
```

这条命令会在 NX 本机桌面会话 `:1` 上打开官方深度画面窗口。

如果当前桌面会话不是 `:1`，可以手动指定：

```bash
ros2 launch robot_bringup depth_camera_view.launch.py display:=:0
```

## 直接运行官方深度查看器

不通过 ROS 也可以直接运行：

```bash
DISPLAY=:1 XAUTHORITY=/home/nx/.Xauthority \
/opt/OrbbecSDK_v1.10.8/Example/bin/depth_viewer
```

## 常见问题

### 只有黑窗口或没有画面

- 先确认没有旧的 Orbbec 进程占用设备：

```bash
pkill -f hello_orbbec || true
pkill -f depth_viewer || true
pkill -f OrbbecViewer || true
```

- 再重新启动 `depth_viewer`。

### `QuickStart` 启动失败

这台 `Astra Pro` 当前会报彩色流模式匹配失败，典型报错是：

```text
Start stream failed, OB_SENSOR_COLOR Match openni video mode failed!
```

这不是深度流本身打不开，而是默认样例同时请求彩色流导致的。请改用 `depth_viewer`。

### 通过 SSH 看不到窗口

- 这个窗口显示在 NX 本机桌面，不会显示在纯文本 SSH 终端里。
- 需要在 NX 接显示器，或者使用带图形桌面的远程方式查看。
